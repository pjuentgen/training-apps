üåü Overview

This Spring Boot application provides various API endpoints that demonstrate different response behaviors and database interactions. Each API call is logged to an H2 in-memory database.

Features:

  • Different response times (fast, normal, slow)
  • Error simulation
  • Random behavior patterns
  • Complete audit trail in database
  • Real-time statistics

http://localhost:8080

GET /api/random

Description

Exhibits random behavior - could be fast, slow, or result in an error. Behavior is determined randomly on each call.

Behavior Response Time Description

Slow 0.5-1.5s Moderate delay
Error Variable Random runtime exception

GET /api/stats

Description

Returns comprehensive statistics about all API calls, including counts by type and recent call history.

Response Fields

Field Type Description
totalCalls number Total number of API calls made
normalCalls number Count of normal API calls
errorCalls number Count of error API calls
fastCalls number Count of fast API calls
slowCalls number Count of slow API calls
randomCalls number Count of random API calls
recentCalls array Last 10 API calls with details
executionTimeMs number Time taken to execute this stats call

üóÑÔ∏è Database Information

The application uses an H2 in-memory database to store API call logs. You can access the database console to view and query the data directly.

Database Access

Console URL http://localhost:8080/h2-console
JDBC URL jdbc:h2:mem:testdb
Username sa
Password password

Database Schema

CREATE TABLE api_calls ( id BIGINT AUTO_INCREMENT PRIMARY KEY, endpoint VARCHAR(255), call_type VARCHAR(50), response_message VARCHAR(500), execution_time_ms BIGINT, created_at TIMESTAMP );

Example Queries

-- Get all API calls SELECT * FROM api_calls ORDER BY created_at DESC; -- Count calls by type SELECT call_type, COUNT(*) as count FROM api_calls GROUP BY call_type; -- Average execution time by endpoint SELECT endpoint, AVG(execution_time_ms) as avg_time FROM api_calls GROUP BY endpoint;
